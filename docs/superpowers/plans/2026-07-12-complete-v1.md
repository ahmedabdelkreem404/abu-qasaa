# Abu Qasaa V1 Completion Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Complete the approved Abu Qasaa V1 scope across Dates merchandising, Real Estate, Services/RFQ, Reports/Audit, and Production Readiness.

**Architecture:** Keep the existing Laravel 12 + Next.js + MySQL modular monolith. Add focused module files under `backend/app/Modules`, additive migrations, scoped APIs, App Router pages, tests, seed data, and documentation. Preserve existing local Google Fonts removal and Phase 10 merchandising work.

**Tech Stack:** Laravel 12, Sanctum, Eloquent, PHPUnit/Pest as configured, Laravel storage, Next.js App Router, TypeScript, Tailwind CSS, MySQL.

## Global Constraints

- Work on branch `codex/complete-v1`.
- Preserve legitimate local work in `frontend/src/app/layout.tsx`, `frontend/src/app/globals.css`, and untracked Phase 10 merchandising files.
- Do not run `git reset --hard`, `git clean -fd`, or destructive commands that delete untracked work.
- Use one backend, one frontend, one database, Modular Monolith, Clean Architecture, and business-unit scoping.
- Do not add microservices, Kubernetes, complex DevOps, AI features, native apps, marketplace, subscriptions, loyalty, full ERP accounting, manufacturing/BOM, or excluded future features.
- Run regression tests after every phase and do not claim completion for a failing phase.
- Use separate focused commits for each phase.

## File Structure

- `backend/app/Modules/Catalog`: Phase 10 Domain enums, Application actions, Infrastructure models, Presentation requests/resources/controllers/routes for merchandising.
- `backend/database/migrations/2026_07_11_000000_create_dates_store_merchandising_tables.php`: preserve and reconcile existing Phase 10 migration.
- `backend/app/Modules/Commerce`: cart/order metadata snapshot changes for bundles.
- `frontend/src/app/(business)/[businessSlug]`: public collections, gift boxes, corporate gifts, seasonal pages, and updated product/cart views.
- `frontend/src/app/(dashboard)/dashboard/catalog/merchandising`: dashboard merchandising management.
- `backend/app/Modules/RealEstate`: Phase 11 module implementation.
- `frontend/src/app/(business)/[businessSlug]/real-estate`: public real-estate pages.
- `frontend/src/app/(dashboard)/dashboard/real-estate`: dashboard real-estate management.
- `backend/app/Modules/ServicesRfq`: Phase 12 module implementation.
- `frontend/src/app/(business)/[businessSlug]/services`, `frontend/src/app/(business)/[businessSlug]/rfq`: public services/RFQ pages.
- `frontend/src/app/(dashboard)/dashboard/services-rfq`: dashboard services/RFQ management.
- `backend/app/Modules/Audit` and `backend/app/Modules/Reports`: Phase 13 logging, reports, exports, and routes.
- `frontend/src/app/(dashboard)/dashboard/reports` and `frontend/src/app/(dashboard)/dashboard/audit-logs`: reporting UI.
- `backend/app/Modules/Core` or existing shared HTTP layer: Phase 14 API error responses, health endpoint, media/upload helpers, rate limits, and security utilities.
- `frontend/src/app`: error boundaries, forbidden/not-found pages, metadata, sitemap, robots, RTL/LTR and production font safety.
- `docs`: roadmap reconciliation, deployment, security, operations, V1 audit, and acceptance checklist.
